## Design Philosophy
- Every screen must feel like a million-euro brand
- Prioritize clarity over visual complexity
- Focus on structure, readability and usability
- Every interface should feel deliberate and consistent
- Avoid unnecessary elements, keep layouts clean
- Design must support function, not distract from it

## Development Principles
- Write clean, maintainable and scalable code
- Prefer simple solutions over complex abstractions
- Ensure performance and fast load times
- Build with long-term extensibility in mind
- Avoid unnecessary dependencies

## UX Principles
- Interfaces must be intuitive without explanation
- Reduce friction wherever possible
- Keep user flows short and clear
- Ensure consistency across all pages and components




## Commit Messages
- NEVER WRITE CLAUDE AS CONTRIBUTOR!
- Use conventional commits: `type(scope): message`
- Types: `feat`, `fix`, `style`, `refactor`, `chore`, `docs`
- Always lowercase
- Max 48 characters
- as short as possible
- No period at the end

Examples:
feat(feed): add tiktok-style vertical scroll
fix(auth): handle phone verification timeout
style(tabbar): update active icon color to accent
refactor(colors): move hardcoded values to constants
chore(deps): update expo sdk to 52
